# MojOS v0.1 — working brief

*Transient work brief. Lives at repo root, deleted when v0.1 ships. The full goal
and finish line live in the thinking repo
([roadmap](https://github.com/mojo-labs-circus/mojo/blob/main/roadmap.md)).*

## Goal

By 15 September 2026: MojOS is Clarke's daily driver — laptop first (all summer),
PC (dual-boot with Windows) the day he's back at uni. mojo-agent runs inside it as
a resident service.

This repo's share of the finish line:

1. The entire system is declared in the flake — apps, Hyprland, dotfiles — and
   rebuilds with one command; the same config boots as a VM.
2. At least two modes exist and switch cleanly: looks (palette, wallpaper, layout),
   behaviour, and how much of the system shows. Candidates: coding mode, Grateful
   Dead mode, normal mode for company.
3. mojo-agent is installed by the flake (as a NixOS module), not by hand.
4. Installed on the laptop as daily driver, replacing Arch — only after the VM
   version is somewhere Clarke would genuinely live.

## Sequence

1. **Learn + skeleton (July).** Nix language basics, flakes, modules, home-manager.
   First flake: boots a minimal Hyprland system in a VM. Multi-host layout from the
   first commit (laptop + pc hosts, shared modules).
2. **The rice + modes (July).** Stylix for system-wide theming; the mode mechanism
   designed by Clarke (specialisations vs home-manager profiles vs switch script —
   decide after understanding the trade-offs). Old dotfiles are reference only —
   the rice is being rebuilt, not ported.
3. **Metal (August).** Partition plan for the laptop, install, daily-drive,
   iterate. Agent module lands here (the agent itself is mojo-agent's brief).
4. **PC (15 September).** New host entry, dual-boot alongside Windows, deploy.

## Open design questions (Clarke decides, in order of arrival)

- Flake layout — study 2–3 well-regarded config repos before committing.
- Mode mechanism — what switches at the Nix level vs at runtime.
- What the agent needs from the OS as a module (service unit, permissions, config
  path) — designed together with mojo-agent.
- Laptop partition/dual-boot story at install time.

## Done when

Points 1–4 above hold on the laptop, and the PC install on 15 September is a
config change plus an afternoon — not a project.
