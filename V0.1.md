# MojOS v0.1 — working brief

*Transient work brief. Lives at repo root, deleted when v0.1 ships. The full goal
and finish line live in the thinking repo
([roadmap](https://github.com/mojo-labs-circus/mojo/blob/main/roadmap.md)).*

## Goal

**Locked 2026-07-05 (see thinking repo's devlog):** mojo-agent proper isn't being
built yet. v0.1's job is a minimal NixOS flake, VM-proven, built specifically to run
[Hermes Agent](https://github.com/NousResearch/Hermes) (self-hosted, MIT) as the
first Jarvis — wholesale, not a hand-built agent core. Once Hermes is running,
*use it* to help design and build the rest of MojOS (modes, tooling, the rice)
rather than hand-building all of that first and adding the agent at the end.

No hard feature-complete deadline attached to this. 15 September is still real as a
*hardware* fact — Clarke's away from his PC until then, laptop is the summer
machine, PC becomes the always-on host when he's back — but it's not a checklist
due by that date. Progress is "is Jarvis running and getting used."

This repo's share, in order:

1. The smallest possible flake that boots in a VM (`nixos-rebuild build-vm`) and
   can run a resident service. Not modes, not the rice yet.
2. Hermes Agent installed on top as a NixOS module/service.
3. From there, daily-driver tooling — screenshots, volume/media keys, clock,
   calendar, clipboard, notifications, a keybind baseline — the "obvious things
   nobody thinks about until they're missing" (macOS/Windows are worth stealing
   from here, not just tiling-WM convention). Built *with* Hermes's help once it's
   running, not before.
4. At least two modes, once the above is solid: looks (palette, wallpaper, layout),
   behaviour, how much of the system shows. Candidates: coding mode, Grateful Dead
   mode, normal mode for company.
5. Installed on the laptop as daily driver, replacing Arch — only after the VM
   version is somewhere Clarke would genuinely live.
6. PC entry (15 September): new host, dual-boot alongside Windows, deploy — should
   be a config change and an afternoon by this point, not a project.

## Open design questions (Clarke decides, in order of arrival)

- Flake layout — study 2–3 well-regarded config repos before committing.
- What Hermes needs from the OS as a module (service unit, permissions, config/
  memory-store path, how it gets network access if it needs any).
- Daily-driver tooling stack — which specific tools per category (screenshot,
  clipboard, launcher, notification daemon, etc.), decided as each is reached.
- Mode mechanism — what switches at the Nix level vs at runtime.
- Laptop partition/dual-boot story at install time.

## Done when

Livable in a VM: boots, Hermes is running and reachable, and the daily-driver
basics (1–3 above) are covered. Modes and metal follow once that's genuinely true.
